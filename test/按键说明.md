## VR UDP 接收和手柄按键测试

测试脚本：

```bash
conda activate light_tp
PYTHONDONTWRITEBYTECODE=1 python test/recv_vr_udp.py \
    --host 10.1.42.3 \
    --port 8080 \
    --print-every 1
```

停止：

```text
Ctrl+C
```

正常收到包时会打印类似：

```text
[packet #1620] from 10.1.42.9:43610 bytes=253 length=62 fps=0.0
  torque=0.00 left_gripper=0.00 right_gripper=0.00
  left_controller   pos=(+0.4291, +0.8280, +0.3986) quat=(+0.3168, +0.0558, +0.3655, +0.8734)
  right_controller  pos=(-0.5749, +0.8361, -0.0949) quat=(+0.1602, +0.2321, +0.4236, +0.8608)
  headset           pos=(-0.7089, +0.8700, +0.4719) quat=(+0.2453, -0.6632, +0.6728, +0.2176)
  left_input        trigger=0.000 grip=0 primary=0 secondary=0 menu=0 joy=(+0.000, +0.000)
  right_input       trigger=0.000 grip=0 primary=0 secondary=0 menu=0 joy=(+0.000, +0.000)
```

字段说明：

```text
left_controller/right_controller: 左右手柄位姿
headset: 头显位姿
left_input/right_input: 左右手柄输入
left_gripper/right_gripper: 机器人夹爪角度字段，不是手柄 grip 按键
```

每只手柄 input 字段顺序：

```text
trigger      扳机，float，一般 0.0 到 1.0
grip         侧边握持键，bool
primary      主按键，bool
secondary    副按键，bool
menu         菜单键，bool
joy          摇杆，joystick_x/joystick_y
```

当前按键映射：

| 打印字段 | 左手柄 | 右手柄 | 备注 |
| --- | --- | --- | --- |
| `trigger` | 左扳机 | 右扳机 | 模拟量，扣下后数值变大 |
| `grip` | 左握持键 | 右握持键 | 后续用于 Grip 相对遥操作使能 |
| `primary` | X | A | 文档已固定 |
| `secondary` | Y | B | 按常见 VR/OpenXR 命名推断，建议实测确认 |
| `menu` | 菜单键 | 菜单键/系统键映射 | 不同发送端可能不同，建议实测确认 |
| `joy` | 左摇杆 | 右摇杆 | `x` 左右方向，`y` 上下方向 |

推荐按键测试顺序：

```text
1. 不按任何键，确认 trigger=0、grip=0、primary=0、secondary=0、menu=0、joy=(0,0)
2. 按左 X，确认 left_input primary=1
3. 按右 A，确认 right_input primary=1
4. 按左 Y，确认 left_input secondary=1
5. 按右 B，确认 right_input secondary=1
6. 扣左扳机，确认 left_input trigger 从 0.000 增大
7. 扣右扳机，确认 right_input trigger 从 0.000 增大
8. 捏左握持键，确认 left_input grip=1
9. 捏右握持键，确认 right_input grip=1
10. 推左摇杆，确认 left_input joy 变化
11. 推右摇杆，确认 right_input joy 变化
12. 按菜单键，确认 menu 字段由 0 变 1，并记录是左手还是右手生效
```

测试时一个键一个键按，观察对应字段是否从 `0` 变成 `1`，或者数值是否变化。
如果同时按多个键，日志里多个字段会同时变化，不方便确认映射。
