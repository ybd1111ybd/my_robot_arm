## VR 遥操作问题记录

### 1. 手柄链路是否正常

先不要直接跑 demo，先确认 UDP 包能收到：

```bash
cd /home/ybd/my_robot
python3 test/recv_vr_udp.py --host 0.0.0.0 --port 8080
```

正常时会看到：

```text
[packet #...] from 10.1.42.5:...
```

如果收不到：

- `ping 10.1.42.5` 能通，只说明头显在线，不代表 VR 程序在发 UDP。
- `sudo tcpdump -ni eth0 host 10.1.42.5` 能看到 ICMP 但没有 UDP，说明头显端发送程序没有发包。
- WSL 当前 IP 是 `10.1.42.44`，头显发送目标应为 `10.1.42.44:8080`。

### 3. 关节运动惩罚

问题：IK 有多解时，某些点会出现关节绕大圈、肩部突然甩动。

解决方向：增加“关节每一帧不要动太多”的代价项：

```bash
--joint-motion-cost 0.10
--joint-motion-cost-profile shoulder-heavy
```

含义：

- `joint-motion-cost`：惩罚当前帧相对上一帧的关节变化。
- `shoulder-heavy`：对肩部关节惩罚更重，减少大臂乱甩。

代码位置：

- `light_teleop/light_teleop/pink_solver.py`
  - `motion_task = PostureTask(...)`
  - 每帧 `motion_task.set_target(self.configuration.q)`
- `light_teleop/light_teleop/vr_visual_demo.py`
  - `--joint-motion-cost`
  - `--joint-motion-cost-profile`
  - `joint_motion_cost_vector(...)`

测试过的结论：

| 配置 | 现象 |
|---|---|
| `0.0` | 关节运动更大，肩部容易打满 |
| `0.03` | 有一点改善，但不明显 |
| `0.10` | 当前比较均衡，肩部峰值下降，末端误差也较小 |
| `0.20` | 肩部更稳，但跟随误差可能变大 |

目前推荐：

```bash
--joint-motion-cost 0.10
--joint-motion-cost-profile shoulder-heavy
```

### 4. 指标怎么看

demo 停止时会输出：

```text
joint_step_l2_mean
joint_step_l2_max
joint_step_abs_max
shoulder_step_abs_max
joint_vel_abs_max
shoulder_vel_abs_max
final_left_error
final_right_error
```

含义：

- `final_left_error/final_right_error`：末端最终误差，越小越跟手。
- `joint_step_l2_mean`：平均每帧整体关节运动量，越小越平滑。
- `joint_step_l2_max`：最大单帧整体跳动，越小越少突然跳。
- `joint_step_abs_max`：单个关节最大单帧动作。
- `shoulder_step_abs_max`：肩部关节最大单帧动作。
- `joint_vel_abs_max`：单个关节最大速度。
- `shoulder_vel_abs_max`：肩部最大速度。

如果 `joint_step_abs_max=0.10000` 或 `shoulder_step_abs_max=0.05000`，说明关节已经打到当前单步限制。

### 5. 关节像被锁死的问题

当前默认姿态模式是：

```bash
--target-orientation-mode wrist-decoupled
```

它的代码在：

```text
light_teleop/light_teleop/vr_visual_demo.py
```

关键逻辑：

```python
decoupled_rotation = pin.exp3(np.array([up_down, left_right, 0.0]))
```

这里第三个值写死成 `0.0`，所以目标方块不能绕局部 z 轴自转。也就是说，绕 z 轴旋转不是 IK 解不出来，而是目标姿态里被主动锁掉了。

如果要测试完整手柄姿态，包括绕 z 轴旋转，用：

```bash
--target-orientation-mode vr
```

例如：

```bash
./light_teleop/scripts/run_vr_demo.sh 10.1.42.44 8080 \
  --left-ee-frame left_arm_link7 \
  --right-ee-frame right_arm_link7 \
  --tcp-control-offset 0.0 \
  --vr-debug-target-forward-offset 0.0 \
  --target-orientation-mode vr \
  --orientation-cost 0.8 \
  --joint-motion-cost 0.10 \
  --joint-motion-cost-profile shoulder-heavy
```

注意：

- `wrist-decoupled`：只保留上下/左右两个姿态通道，z 轴自转锁死。
- `vr`：完整使用手柄姿态，目标方块可以绕 z 轴旋转。
- `locked`：完全锁定姿态。
- `swing`：只对齐方向轴，不适合测试自转。
