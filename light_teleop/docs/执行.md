# light_teleop 执行方案与接口说明

## 1. 文档目的

本文档说明 `light_teleop` 实现时使用的技术、依赖、数据结构、模块接口和
运行方式。它回答的问题是：

```text
用什么技术做？
每个模块怎么调用？
数据在模块之间怎么传？
第一阶段怎么运行？
后续下发接口怎么预留？
```

本文档对应 [architecture.md](./architecture.md) 和 [plan.md](./plan.md)。

## 2. 技术栈

### 2.1 主要语言

第一版使用 Python。

原因：

- Pink 本身是 Python 库。
- Pinocchio/Pink/Meshcat 组合在 Python 中使用最直接。
- 先做可视化和 IK 验证时，Python 迭代速度更快。
- 避免把 Pink 嵌进原版 C++ ROS 2 节点导致架构变重。

### 2.2 核心依赖

| 技术 | 用途 |
| --- | --- |
| Python 3.10+ | 主开发语言 |
| Pinocchio | 从 URDF 构建机器人运动学模型 |
| Pink | 微分逆运动学结算 |
| qpsolvers | Pink 底层 QP 求解器接口 |
| numpy | 数值计算 |
| Meshcat | 浏览器可视化 |
| meshcat-shapes | pip 包名；Python 中通常 `import meshcat_shapes`，用于显示 target frame 和辅助几何 |
| loop-rate-limiters | 控制主循环频率 |
| socket | UDP 手柄数据接收 |
| dataclasses | 定义轻量数据结构 |

### 2.3 可选依赖

| 技术 | 用途 |
| --- | --- |
| ROS 2 rclpy | 后续发布 ROS 2 topic |
| sensor_msgs | 后续发布 `JointState` |
| std_msgs | 后续发布数组命令 |
| scipy | 如果需要更方便的四元数和旋转工具 |
| pytest | 单元测试 |

第一阶段不强制依赖 ROS 2。

当前控制范围只包含双臂 14 个关节：

```text
left_joint1..left_joint7
right_joint1..right_joint7
```

腰、头、夹爪不进入 Pink IK。夹爪开合由独立链路控制；UDP 中夹爪字段只做
兼容解析和调试保留。

## 3. 外部资产和路径

### 3.1 URDF

默认使用：

```text
jz_descripetion-main/robot_urdf/urdf/robot urdf.10.8.SLDASM.urdf
```

说明：`jz_descripetion-main` 是当前仓库目录名，虽然拼写不是 `description`，
实现时不要擅自改路径。

### 3.2 Meshes

默认使用：

```text
jz_descripetion-main/robot_urdf/meshes/visual
jz_descripetion-main/robot_urdf/meshes/collision
```

当前 URDF 存在已知缺失 mesh 引用，主要是夹爪相关 STL：

```text
gripper_base_link.STL
narrow1_Link.STL
narrow2_Link.STL
narrow3_Link.STL
wide1_Link.STL
wide2_Link.STL
wide3_Link.STL
```

这些文件在 `meshes/visual` 和 `meshes/collision` 中都可能缺失。第一阶段实现
必须支持：

```text
运动学模型加载成功是硬要求。
完整夹爪几何显示不是 M1-M3 硬要求。
缺失 mesh 必须被明确记录为 warning。
```

### 3.3 package:// 解析

URDF 使用：

```text
package://jz_robot_description/...
```

包名来自：

```text
jz_descripetion-main/robot_urdf/package.xml
```

包路径是：

```text
jz_descripetion-main/robot_urdf
```

实现时必须通过加载验证脚本确认 `package://jz_robot_description/meshes/...`
能解析到当前仓库中的真实文件。`package_dirs` 应指向能让 Pinocchio 找到
`jz_robot_description` 包的搜索根；不要只传 `meshes` 目录。

### 3.4 Pink 参考代码

参考目录：

```text
reference/pink-main
```

项目实现时优先使用安装后的 `pin-pink` 包。如果暂时不安装，也可以把
`reference/pink-main` 加入 `PYTHONPATH` 做开发验证。

## 4. 运行模式

### 4.1 simulated 模式

不用 VR 手柄，程序内部生成左右 target。

用途：

- 验证 URDF 加载。
- 验证 Meshcat 显示。
- 验证 Pink 双臂 IK。
- 调整末端 frame 和任务权重。

数据流：

```text
simulate target
  -> pink_solver
  -> visualizer
```

### 4.2 vr_visual 模式

使用 VR 手柄，但只做可视化，不下发真机。

用途：

- 验证 UDP 接收。
- 验证手柄解析。
- 验证 Grip 相对控制。
- 验证手柄控制手感。

数据流：

```text
vr_receiver
  -> vr_parser
  -> teleop_mapper
  -> pink_solver
  -> visualizer
```

### 4.3 dry_run_publish 模式

使用 Pink 输出的 `IkResult` 生成 dry-run 关节命令，只打印或记录，不连接真机。

用途：

- 验证下发格式。
- 验证关节名顺序。
- 验证下发频率。

数据流：

```text
pink_solver
  -> command_publisher(dry_run=True)
```

### 4.4 robot_publish 模式

后续真机模式。第一版不启用。

数据流：

```text
pink_solver
  -> safety layer
  -> command_publisher
  -> robot controller
```

## 5. 核心数据结构

第一版建议用 Python dataclass 定义模块之间传递的数据，避免传散乱 dict。

### 5.1 Pose

表示普通位姿。

```python
@dataclass
class Pose:
    position: np.ndarray  # shape: (3,), [x, y, z]
    quaternion: np.ndarray  # shape: (4,), [qx, qy, qz, qw]
```

约定：

- position 单位为米。
- quaternion 顺序为 `[qx, qy, qz, qw]`。
- 传入 Pink/Pinocchio 前转换为 `pin.SE3`。

### 5.2 ControllerInput

表示手柄输入。

```python
@dataclass
class ControllerInput:
    trigger: float
    grip_button: bool
    primary_button: bool
    secondary_button: bool
    menu_button: bool
    joystick_x: float
    joystick_y: float
```

### 5.3 ControllerState

表示单只手柄状态。

```python
@dataclass
class ControllerState:
    pose: Pose
    input: ControllerInput
```

### 5.4 VrPacket

表示完整 VR 数据包。

```python
@dataclass
class VrPacket:
    torque: float
    angles: np.ndarray  # shape: (16,)
    left_controller: ControllerState
    right_controller: ControllerState
    headset_pose: Pose
    has_vr_device_poses: bool
    has_controller_inputs: bool
```

### 5.5 TeleopTargets

表示映射后的机器人目标。

```python
@dataclass
class TeleopTargets:
    left_target: pin.SE3
    right_target: pin.SE3
    left_enabled: bool
    right_enabled: bool
```

### 5.6 IkResult

表示 Pink solver 的输出。它属于 IK 层，不包含任何下发格式。

```python
@dataclass
class IkResult:
    q: np.ndarray
    left_joint_names: list[str]
    left_positions: np.ndarray
    right_joint_names: list[str]
    right_positions: np.ndarray
    left_ee_pose: pin.SE3
    right_ee_pose: pin.SE3
```

### 5.7 JointCommand

表示后续下发命令。

```python
@dataclass
class JointCommand:
    stamp: float
    left_joint_names: list[str]
    left_positions: np.ndarray
    right_joint_names: list[str]
    right_positions: np.ndarray
    mode: str  # "position" or "velocity"
```

`JointCommand` 只能由 `command_publisher.py` 或下发适配层生成，不能由
`pink_solver.py` 直接返回。

```python
    robot: pin.RobotWrapper,
    left_joint_names: list[str],
    right_joint_names: list[str],
    left_ee_frame: str,
    right_ee_frame: str,
) -> None:
    ...
```

模块输出：

```text
pin.RobotWrapper
```

失败条件：

- URDF 不存在。
- `package://jz_robot_description/...` 无法解析。
- 缺少左右臂关节。
- 缺少末端 frame。

降级条件：

- 只有夹爪或非关键 visual/collision mesh 缺失时，不阻塞运动学模型加载。
- 缺失 mesh 必须输出清单。
- 若缺失 mesh 导致 visual model 构建失败，实现可先构建 kinematic-only
  模型，并在可视化中只显示 frame 和可用几何。

### 6.2 pink_solver.py

建议类：

```python
class PinkTeleopSolver:
    def __init__(
        self,
        robot: pin.RobotWrapper,
        left_ee_frame: str,
        right_ee_frame: str,
        left_joint_names: list[str],
        right_joint_names: list[str],
        solver: str,
    ) -> None:
        ...

    def set_targets(
        self,
        left_target: pin.SE3,
        right_target: pin.SE3,
    ) -> None:
        ...

    def step(self, dt: float) -> IkResult:
        ...

    def get_configuration(self) -> pink.Configuration:
        ...

    def get_current_ee_poses(self) -> tuple[pin.SE3, pin.SE3]:
        ...
```

输入：

```text
left_target: pin.SE3
right_target: pin.SE3
dt: float
```

输出：

```text
IkResult
```

内部使用：

```text
pink.Configuration
pink.tasks.FrameTask
pink.tasks.PostureTask
pink.solve_ik
```

调用形式：

```python
velocity = solve_ik(configuration, tasks, dt, solver=solver)
configuration.integrate_inplace(velocity, dt)
```

solver 默认选择：

```python
if "daqp" in qpsolvers.available_solvers:
    solver = "daqp"
else:
    solver = qpsolvers.available_solvers[0]
```

如果 `qpsolvers.available_solvers` 为空，程序应在启动时报错退出。

关节范围：

- `PinkTeleopSolver` 只管理双臂 14 个关节。
- 推荐输入模型是 reduced model。
- 如果使用 full model，必须显式锁定腰、头、夹爪等非双臂关节。
- `PostureTask` 只是正则项，不等价于锁定非双臂关节。

### 6.5 vr_parser.py

建议接口：

```python
def parse_vr_packet(data: bytes) -> VrPacket | None:
    ...
```

解析协议：

```text
header: 0xAA 0xBB
length: uint8, 原版 4 字节单位数，不是 packet byte length
torque: int16 little-endian, scaled by / 10.0
angles: 16 * float32 little-endian
left_controller_pose: 3 float32 position + 4 float32 quaternion
right_controller_pose: 3 float32 position + 4 float32 quaternion
headset_pose: 3 float32 position + 4 float32 quaternion
left_input: trigger float32 + 4 buttons + joystick_x/y float32
right_input: trigger float32 + 4 buttons + joystick_x/y float32
```

length 字段语义必须按原 C++ 实现兼容：

```text
declared_total_size = 2 + 1 + length * 4
```

其中：

```text
旧包 length = 16
新包 length = 62
```

注意：旧包 `declared_total_size = 67`，但实际最小包为 69 bytes，因为 torque
额外占 2 bytes。这是原协议历史包袱。Python parser 不能把 length 当作真实
packet byte length，只能把它作为兼容校验字段。

最小包：

```text
69 bytes
```

完整包：

```text
185 bytes
```

异常返回：

```text
None
```

### 6.6 teleop_mapper.py

建议类：

```python
class TeleopMapper:
    def __init__(
        self,
        axis_mapping: list[int],
        scale_factor: float,
        left_home_target: pin.SE3,
        right_home_target: pin.SE3,
    ) -> None:
        ...

    def update(
        self,
        packet: VrPacket | None,
        current_left_ee: pin.SE3,
        current_right_ee: pin.SE3,
    ) -> TeleopTargets:
        ...
```

输入：

```text
VrPacket | None
current_left_ee
current_right_ee
```

输出：

```text
TeleopTargets
```

关键交互：

- left grip 控制左臂。
- right grip 控制右臂。
- left primary button 对应 X。
- right primary button 对应 A。
- primary button 设置对应或双侧 home target，具体策略在实现前固定。

建议第一版策略：

```text
任一 primary button 按下时，左右 target 都回 home。
```

这样和原版行为接近，也便于调试。

`packet is None` 时：

```text
不改变 Grip 状态机。
返回上一帧 target。
如果还没有上一帧 target，则返回 home target。
不触发回零、不触发锁定、不让 target 跳变。
```

姿态映射：

```text
不能直接交换四元数分量。
必须使用旋转矩阵/坐标基变换来映射 VR delta rotation。
必须支持左右手 tool_orientation_offset。
```

姿态映射测试：

```text
VR 绕 X/Y/Z 单轴旋转，机器人末端姿态方向符合预期。
左手和右手分别测试。
四元数输出归一化。
四元数符号翻转不会造成 target 跳变。
```

### 6.7 command_publisher.py

建议类：

```python
class CommandPublisher:
    def __init__(self, enabled: bool = False, dry_run: bool = True) -> None:
        ...

    def make_command(self, ik_result: IkResult) -> JointCommand:
        ...

    def publish(self, command: JointCommand) -> None:
        ...

    def close(self) -> None:
        ...
```

第一版行为：

```text
enabled = False: 什么都不做
dry_run = True: 打印或记录命令
dry_run = False: 后续接真机或 ROS 2
```

边界约束：

- `CommandPublisher` 接收 `IkResult` 并转换为 `JointCommand`。
- `PinkTeleopSolver` 不知道 ROS 2 topic、SDK 或数组命令格式。
- `JointCommand` 只包含双臂 14 个关节，不包含夹爪、腰、头。

## 7. 主循环接口

主循环建议频率：

```text
100 Hz
```

伪代码：

```python
robot = load_robot_model(config.urdf_path, config.package_dirs)
validate_robot_model(...)

solver = PinkTeleopSolver(...)
visualizer = TeleopVisualizer(robot)
mapper = TeleopMapper(...)
receiver = UdpVrReceiver(...) if mode uses vr else None
publisher = CommandPublisher(enabled=False, dry_run=True)

while running:
    if mode == "simulated":
        targets = simulated_targets(...)
    else:
        data = receiver.receive_once()
        packet = parse_vr_packet(data) if data else None
        current_left, current_right = solver.get_current_ee_poses()
        targets = mapper.update(packet, current_left, current_right)

    solver.set_targets(targets.left_target, targets.right_target)
    ik_result = solver.step(dt)

    left_pose, right_pose = solver.get_current_ee_poses()
    visualizer.display_robot(solver.get_configuration().q)
    visualizer.display_targets(targets.left_target, targets.right_target)
    visualizer.display_current_ee(left_pose, right_pose)

    command = publisher.make_command(ik_result)
    publisher.publish(command)
    rate.sleep()
```

## 8. Pink 任务配置

第一版建议任务：

```text
left_ee_task
right_ee_task
posture_task
```

参与 IK 的关节：

```text
只允许 left_joint1..7 和 right_joint1..7。
夹爪、腰、头固定。
```

建议初始权重：

```text
left/right position_cost = 1.0
left/right orientation_cost = 0.5
posture cost = 1e-3
lm_damping = 1.0
```

如果出现姿态乱折：

- 增大 `posture_task`。
- 降低 orientation cost。
- 调整初始 q。
- 确认是否使用 reduced model，避免腰/头/夹爪参与 IK。

如果末端跟随慢：

- 增大 frame task cost。
- 检查 QP solver。
- 检查速度限制。
- 检查目标是否跳变过大。

## 9. 坐标映射接口

沿用原版推荐映射：

```text
axis_mapping = [-2, 3, 1]
```

表示：

```text
VR x -> robot -y
VR y -> robot +z
VR z -> robot +x
```

建议接口：

```python
def apply_axis_mapping(
    vector: np.ndarray,
    axis_mapping: list[int],
) -> np.ndarray:
    ...
```

输入：

```text
vector shape = (3,)
axis_mapping values in {-3, -2, -1, 1, 2, 3}
```

输出：

```text
mapped vector shape = (3,)
```

验收：

- 每个轴只能出现一次。
- 0 不允许。
- 绝对值必须在 1 到 3。

### 9.1 姿态映射接口

建议接口：

```python
def map_rotation_delta(
    vr_delta_quat: np.ndarray,
    axis_mapping: list[int],
    tool_orientation_offset: pin.Quaternion | np.ndarray,
) -> np.ndarray:
    ...
```

实现要求：

```text
1. 将 vr_delta_quat 转成旋转矩阵。
2. 按 axis_mapping 做坐标基映射或复刻原 C++ 行列映射算法。
3. 转回四元数并归一化。
4. 应用左右手 tool_orientation_offset。
5. 返回 [qx, qy, qz, qw]。
```

不允许：

```text
直接交换四元数 x/y/z 分量。
忽略左右手 tool offset。
```

## 10. UDP 接口约定

默认监听：

```text
host = 0.0.0.0
port = 8080
```

完整包字段：

| 字段 | 类型 | 数量 | 说明 |
| --- | --- | --- | --- |
| header | uint8 | 2 | `0xAA 0xBB` |
| length | uint8 | 1 | 4 字节单位数，不是真实包字节长度 |
| torque | int16 | 1 | 小端，除以 10 |
| angles | float32 | 16 | 左臂 7 + 左夹爪 + 右臂 7 + 右夹爪 |
| left pose | float32 | 7 | xyz + qx qy qz qw |
| right pose | float32 | 7 | xyz + qx qy qz qw |
| headset pose | float32 | 7 | xyz + qx qy qz qw |
| left input | mixed | 16 bytes | trigger + 4 buttons + joystick |
| right input | mixed | 16 bytes | trigger + 4 buttons + joystick |

角度字段第一版不用于 Pink 结算，只作为调试和兼容保留。

最小包和完整包：

```text
最小包实际大小: 69 bytes
完整包实际大小: 185 bytes
```

兼容校验：

```text
declared_total_size = 2 + 1 + length * 4
如果 declared_total_size > received_size，则丢包。
如果 length 不是 16 或 62，可以 warning，但仍可继续严格数值校验。
```

## 11. 下发接口约定

第一版不接真机，只定义接口。

关节顺序固定：

```text
left_joint1

left_joint3
left_joint4
left_joint5
left_joint6
left_joint7
right_joint1
right_joint2
right_joint3
right_joint4
right_joint5
right_joint6
right_joint7
```

不包含：

```text
waist_joint_*
head_joint_*
left/right gripper joints
```

输出单位：

```text
radian
```

```text
name: joint names

```

后续如果发布数组命令，必须在文档中固定数组顺序，不能依赖隐式约定。

## 12. 日志和错误处理

第一版使用 Python 标准 `logging`。

日志级别：

```text
INFO: 正常启动、路径、frame、solver、Meshcat URL
WARNING: 丢包、无效输入、target 超限、频率不足
ERROR: URDF 加载失败、缺失关节、solver 失败
```

错误处理原则：

- 输入错误不退出主程序，丢弃当前包。
- 模型加载错误直接退出。
- Pink 求解失败时保持上一帧命令，并打印 warning。
- 真机下发阶段求解失败必须触发保护策略。

## 15. 真机接入前检查清单

真机下发前必须完成：

- URDF 关节方向确认。

- Pink 输出关节顺序确认。

- 关节速度限幅确认。
- 手柄 Grip 使能确认。

## 16. 执行优先级

最高优先级：

```text
model_loader.py
pink_solver.py
visualizer.py
main.py simulated mode
```

第二优先级：

```text
vr_parser.py
vr_receiver.py
teleop_mapper.py
main.py vr_visual mode
```

第三优先级：

```text
command_publisher.py dry-run
ROS 2 publisher
真机下发接口
安全层
```

整体原则是：先让模型动起来，再接手柄；先在屏幕上动得对，再考虑下发。
