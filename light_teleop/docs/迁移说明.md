# light_teleop 迁移说明

更新时间：2026-06-26

本文档给接手者使用。目标是：拿到 `my_robot` 目录的 zip 后，在另一台电脑上恢复当前
`light_teleop` 开发环境，并能跑通模型检查、Meshcat 可视化、Pink IK 模拟和 VR 可视化
demo。

当前阶段仍然是可视化调试链路：

```text
VR UDP 输入
  -> 手柄相对位姿映射
  -> TCP target 生成
  -> Pink 双臂 IK
  -> Meshcat 可视化
```

当前代码不发布真机关节命令，也没有接真机 command publisher。交接验证时只跑可视化和
离线 IK，不要操控真实机器人。

## 4. 创建 conda 环境

本项目约定使用 `light_tp` conda 环境。

```bash
conda create -n light_tp python=3.10 -y
conda activate light_tp

python -m pip install --upgrade pip setuptools wheel
python -m pip install -e ./light_teleop
python -m pip install daqp
```

说明：

- 必须用 `python -m pip install -e ./light_teleop` 做 editable 安装。
- 不建议把 `light_teleop` 做成普通 wheel 后安装，因为 VR demo 会按源码位置寻找
  `test/teleop_vr_py/`。
- `daqp` 是当前已验证的默认 QP solver。代码会优先选 `daqp`；没有 `daqp` 时会选环境里其它
  可用 solver，但迁移时优先保持和当前机器一致。

当前机器已验证的关键版本如下，仅作排查参考，不要求手工逐个固定：

## 8. VR UDP 连接检查

先确认新机器的本机 IP：

```bash
ip addr
```

假设新机器接收 VR 包的 IP 是 `<LOCAL_IP>`，先只检查 UDP 包：

```bash
conda activate light_tp

python test/recv_vr_udp.py \
  --host <LOCAL_IP> \
  --port 8080 \
  --print-every 10
```

如果出现下面这种错误，通常是 `--host` 不是本机网卡 IP：

```text
OSError: [Errno 99] Cannot assign requested address
```

排查方式：

- 把 `--host` 改成 `0.0.0.0` 临时监听所有网卡。
- 或者把 VR 发送端目标 IP 改成新机器真实 IP。
- 确认端口仍是 `8080`。

收到包后，期望终端周期性打印：

```text
[packet #...] from ...
left_controller
right_controller
headset
left_input
right_input
```

也可以跑纯 Python target 映射打印 demo：

```bash
python test/vr_recv_demo.py \
  --host <LOCAL_IP> \
  --port 8080 \
  --print-every 10
```

操作语义：

```text
Grip 按住：锁定当前手柄 pose，从当前 target 开始做相对控制
Grip 松开：停止跟随，保持上一帧 target
左 X / 右 A：回到 home target
```

## 10. 配置调参入口

现场调参优先改：

```text
light_teleop/config/vr_visual_demo.yml
```

命令行参数会覆盖配置文件，例如：

```bash
PYTHONDONTWRITEBYTECODE=1 python -m light_teleop.vr_visual_demo \
  --host <LOCAL_IP> \
  --wrist-left-right-gain -1.15 \
  --wrist-up-down-gain 1.15 \
  --log-file light_teleop/light_teleop/logs/vr_visual_demo.log
```

常用调参方向：

```yaml
# 左右方向反
wrist_left_right_gain: 1.3

# 左右不够灵活
wrist_left_right_gain: -1.5

# 上下不够灵活
wrist_up_down_gain: 1.5

# 翻腕幅度顶住
wrist_max_angle: 1.0

# 整体发飘
wrist_left_right_gain: -1.15
wrist_up_down_gain: 1.15

# TCP 点方向不对
tcp_control_offset: -0.10
```

如果 TCP 点轴向不对，再改：

```yaml
tcp_control_offset_axis: x
```

## 12. 常见问题

### 12.1 `ModuleNotFoundError: No module named light_teleop.model_loader`

通常是没有做 editable 安装，或当前 conda 环境不是 `light_tp`。

处理：

```bash
conda activate light_tp
python -m pip install -e ./light_teleop
```

### 12.2 `ModuleNotFoundError: No module named teleop_vr_py`

`vr_visual_demo.py` 依赖项目根目录下的 `test/teleop_vr_py/`。确认交接包里保留了 `test/`
目录，并且使用的是 editable 安装的源码位置。

### 12.3 `no qpsolvers solver is available`

安装 QP solver：

```bash
conda activate light_tp
python -m pip install daqp
```

然后重新检查：

```bash
python - <<'PY'
import qpsolvers
print(qpsolvers.available_solvers)
PY
```

### 12.4 UDP 绑定失败

错误类似：

```text
Cannot assign requested address
```

说明 `host` 不是本机 IP。把 `light_teleop/config/vr_visual_demo.yml` 的 `host` 改成新机器
IP，或临时用：

```bash
--host 0.0.0.0
```

### 12.5 Meshcat 打不开或页面空白

先确认终端打印了 URL。通常是：

```text
http://127.0.0.1:7000/static/
```

如果 mesh 加载干扰排查，先用骨架模式：

```bash
PYTHONDONTWRITEBYTECODE=1 python -m light_teleop.visualizer \
  --duration 5 \
  --frequency 10 \
  --no-meshes
```

### 12.6 `log_file` 路径报错

当前默认配置已经使用相对路径：

```yaml
log_file: light_teleop/light_teleop/logs/vr_visual_demo.log
```

如果从非项目根目录运行，可能会解析到不合适的位置。处理方式是先 `cd` 到项目根目录，或者运行时加：

```bash
--log-file light_teleop/light_teleop/logs/vr_visual_demo.log
```
